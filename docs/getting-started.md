# Начало работы с Postery

## Требования

- Python 3.12+
- Linux / macOS (на Windows рекомендуется WSL)
- Доступ к серверу или локальная машина

---

## 1. Скачайте проект

```bash
git clone https://github.com/prog-time/postery.git
cd postery
```

---

## 2. Запустите проект

```bash
./start.sh
```

Скрипт автоматически:
- Установит `python3-venv`, если его нет
- Создаст виртуальное окружение `.venv`
- Установит все зависимости
- Создаст файл `.env` из `.env.example`
- Создаст директорию `data/` для базы данных
- Запустит сервер

> Если получаете ошибку `Permission denied` — выполните:
> ```bash
> chmod +x start.sh
> ```

---

## 3. Откройте админ-панель

После запуска откройте браузер и перейдите по адресу:

```
http://localhost:8000/admin
```

По умолчанию создаётся пользователь `admin` с паролем `admin`.

> **Важно:** сразу смените пароль после первого входа (см. раздел ниже).

---

## 4. Создание или смена пользователя

Для создания нового суперадмина или смены пароля выполните в папке проекта:

```bash
source .venv/bin/activate
python create_superadmin.py
```

Скрипт запросит логин и пароль:

```
=== Создание / обновление суперадмина ===

Логин: ваш_логин
Пароль:
```

- Если пользователь с таким логином уже существует — пароль будет обновлён
- Если нет — будет создан новый суперадмин

> Пароль должен быть не короче 6 символов.

---

## 5. Настройка окружения

Файл `.env` создаётся автоматически при первом запуске. Откройте его и настройте:

```bash
nano .env
```

| Переменная | Описание |
|------------|----------|
| `SECRET_KEY` | Секретный ключ сессий — **обязательно замените** в продакшене |
| `DATABASE_URL` | Строка подключения к БД (по умолчанию SQLite) |

Пример для SQLite (по умолчанию):
```
DATABASE_URL=sqlite:///data/admin.db
```

Пример для PostgreSQL:
```
DATABASE_URL=postgresql://user:password@host:5432/dbname
```

---

## 6. Остановка сервера

Нажмите `Ctrl+C` в терминале, где запущен `./start.sh`.
